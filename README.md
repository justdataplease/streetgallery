# Sotyriadromou.gr — Sotyria Dromou, Ζωγραφική & Γλυπτική

Στατική ιστοσελίδα-πορτφόλιο. **Χωρίς build, χωρίς εξαρτήσεις** — σκέτο HTML/CSS/JS,
έτοιμη για GitHub Pages. Για τοπική προεπισκόπηση αρκεί να ανοίξετε το `index.html`
στον browser.

## Δομή

```
index.html          Γκαλερί (masonry) με φίλτρα κατηγοριών
artwork.html        Σελίδα έργου: εικόνα, τίτλος, κατηγορία, ετικέτες, ιστορία
about.html          Βιογραφικό & επικοινωνία
data/artworks.js    ⭐ ΟΛΟ το περιεχόμενο είναι εδώ (ένα μπλοκ ανά έργο)
images/erga/        Οι φωτογραφίες των έργων (βελτιστοποιημένες για web)
content/            Πρωτότυπες φωτογραφίες (ΔΕΝ ανεβαίνουν — είναι στο .gitignore)
CNAME               Το custom domain (Sotyriadromou.gr)
```

## ➕ Πώς προσθέτω νέο έργο (2 βήματα)

1. **Βάλτε τη φωτογραφία** στον φάκελο `images/erga/`, π.χ. `to-neo-ergo.jpg`
   (κατά προτίμηση ως 1400px στη μεγάλη πλευρά — αλλά δουλεύει και χωρίς σμίκρυνση).

2. **Ανοίξτε το `data/artworks.js`** και προσθέστε ένα μπλοκ στην ΑΡΧΗ της λίστας
   (τα πρώτα εμφανίζονται πρώτα):

```js
{
  id: "to-neo-ergo",                    // μοναδικό, λατινικά, χωρίς κενά
  title: "Το νέο έργο",
  image: "to-neo-ergo.jpg",             // όνομα αρχείου μέσα στο images/erga/
  category: "Καθημερινότητα",           // μία από τις CATEGORIES (πάνω-πάνω στο αρχείο)
  tags: ["ελαιογραφία", "αθήνα"],       // προαιρετικά
  story: `
    Προαιρετική ιστορία του έργου. Γράψτε ελεύθερα —
    μια κενή γραμμή ανάμεσα στις προτάσεις δημιουργεί νέα παράγραφο.

    Αν δεν υπάρχει ιστορία, αφήστε το κενό ή σβήστε τη γραμμή εντελώς.
  `,
},
```

Αποθήκευση → commit → push. Το GitHub Pages ανανεώνει το site αυτόματα σε ~1 λεπτό.
(Μπορείτε να τα κάνετε όλα και μέσα από το site του GitHub: *Add file → Upload files*
για την εικόνα και το μολυβάκι ✏️ για το `data/artworks.js`.)

- **Νέα κατηγορία;** Προσθέστε την στη λίστα `CATEGORIES` στην κορυφή του `data/artworks.js`.
- **Χωρίς ιστορία;** Κανένα πρόβλημα — η σελίδα του έργου δείχνει απλώς την εικόνα και τον τίτλο.

## 🚀 Πρώτο ανέβασμα στο GitHub Pages

1. Δημιουργήστε ένα repository στο GitHub (π.χ. `Sotyriadromou.gr`).
2. Από αυτόν τον φάκελο:
   ```
   git remote add origin https://github.com/<USERNAME>/<REPO>.git
   git push -u origin main
   ```
3. Στο GitHub: **Settings → Pages → Source: Deploy from a branch → Branch: `main` / (root)**.
4. **Domain:** Στο ίδιο σημείο δηλώστε custom domain `Sotyriadromou.gr` (το αρχείο
   `CNAME` υπάρχει ήδη). Στον DNS provider του domain προσθέστε:
   - `A` records για το apex (`Sotyriadromou.gr`) → `185.199.108.153`,
     `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - προαιρετικά `CNAME` για το `www` → `<USERNAME>.github.io`
5. Ενεργοποιήστε το **Enforce HTTPS** μόλις εκδοθεί το πιστοποιητικό.

## 🛠 Τεχνικές σημειώσεις

- Οι εικόνες στο `images/erga/` έχουν περάσει από αυτόματη διόρθωση προσανατολισμού,
  αφαίρεση διπλότυπων και συμπίεση (max 1400px, JPEG q82, progressive).
- Το `content/` κρατά τα πρωτότυπα εκτός repo.
- Δεν χρησιμοποιείται Jekyll (`.nojekyll`) — ό,τι βλέπετε είναι ό,τι σερβίρεται.
